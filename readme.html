Overview

About this website

The Lake Julia Association website is a static HTML site — meaning it is built entirely from plain HTML, CSS, and JavaScript files with no database, no WordPress, no server-side code, and no ongoing software to update. This makes it extremely simple to maintain and very reliable to host.

The site replaces a previous Google Sites page, preserving and expanding on its content. It was designed and built in early 2026 by Scott Schlueter, technology officer of the Association. As of April 2026, the site is hosted on GitHub Pages and the domain is managed through Cloudflare for DNS and HTTPS.

⚠️

Delete Thumbs.db files before deploying

Windows automatically creates Thumbs.db files in image folders. These are harmless but unnecessary. They can be safely deleted from the images folders.

Infrastructure

How the hosting works

The site is hosted on GitHub Pages — a free static site hosting service provided by GitHub. Files are stored in a Git repository and published automatically when changes are pushed. The domain is managed through Cloudflare for DNS and HTTPS. Ongoing costs are essentially zero (just the domain name, ~$10/year).

☁️

No hardware dependency

Unlike the previous Synology NAS setup, the site no longer depends on any hardware at a private residence. GitHub's infrastructure handles uptime, and the site will stay online regardless of power or internet conditions at Scott's home.

If the site goes down

Check githubstatus.com to see if GitHub Pages is experiencing an outage. These are rare and resolve on their own. If the custom domain stops resolving, check the DNS record in Cloudflare — it should be a CNAME pointing to sschlueter86.github.io with proxy status set to DNS only (gray cloud).

Maintenance

Keeping the site up to date

Updating existing content

Since the site is plain HTML, any page can be edited directly with a text editor. The recommended workflow is:

  1. Open your local working copy at D:\LakeJulia_GIS\LakeJulia_stage in VS Code
  2. Find the page you want to edit and make your changes
  3. Preview in a browser locally to verify the changes look correct
  4. In VS Code's Source Control panel (or GitHub Desktop), stage the changed files, write a short commit message describing the change, and push to GitHub
  5. The change is live at lakejuliawi.org within approximately 30 seconds
💻

Using VS Code Claude Code extension for larger edits

Using the VS Code Claude Code extension is highly recommended for larger edits. The extension can be pointed to local files and will match current formats.

Adding a new page

Every page follows the same template structure. To add a new page:

  1. Copy an existing page from the same section (e.g. history/archives.html) and rename it for your new page
  2. Update the <title> tag and breadcrumb at the top
  3. Update the class="active" attribute in the subnav to match the new page
  4. Replace the content in the .content-wrap div
  5. Add a link to the new page in the section's index.html and in js/nav.js if needed

Enabling member-only access

When the site begins hosting private content, Cloudflare Access should be enabled. This adds an email verification gate — visitors enter their email address, receive a one-time code, and are granted access if their email is on the approved list.

Setup steps when ready:

  1. Go to one.dash.cloudflare.com → Access controls → Applications
  2. Click Add an application → Self-hosted
  3. Set domain to lakejuliawi.org
  4. Create a policy with action Allow, rule type Email list
  5. Go to My Team → Lists and create an Email list by uploading a CSV of member emails
  6. Free tier supports up to 50 users
